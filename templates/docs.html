Authentication

Security is paramount at Orgteh. Our API strictly enforces the Bearer Token authentication standard over HTTPS. You must pass your unique API Key in the headers of every request.

Your API Key is a secret credential. Never expose this key in client-side code (browsers) or commit it to public repositories.

Header Structure

Every API call must include the Authorization header in the format:

# HTTP Header Format
Authorization: Bearer Orgteh_ai8x7d6f5g4h...

Security Best Practices

  • Store API keys in environment variables, never in code
  • Use different keys for development and production
  • Rotate keys regularly from your dashboard
  • Never share your API key with anyone

We support key rotation. If you suspect a leak, regenerate your key instantly from the dashboard.

Requests & Responses

Interacting with the Orgteh API involves sending a POST request with a JSON body, primarily managing the messages array for conversation history.

We support advanced features like Streaming (stream: true), which sends Server-Sent Events (SSE) for real-time typing effects.

Python (SDK)
JavaScript (Fetch)
HTTP / cURL
from openai import OpenAI # Configure client with Orgteh Base URL client = OpenAI( base_url="https://orgteh.com/v1", api_key="Orgteh_ai..." ) # Send a streaming request response = client.chat.completions.create( model="deepseek-ai/deepseek-v3.2", messages=[ { "role": "system", "content": "You are a helpful assistant." }, { "role": "user", "content": "Hello world!" } ], temperature=0.7, stream=True # Enable real-time token streaming ) for chunk in response: if chunk.choices[0].delta.content: print(chunk.choices[0].delta.content, end="")
const url = "https://orgteh.com/v1/chat/completions"; // Request Configuration const requestOptions = { method: "POST", headers: { "Content-Type": "application/json", "Authorization": "Bearer Orgteh_ai..." }, body: JSON.stringify({ model: "deepseek-ai/deepseek-v3.2", messages: [ { role: "user", content: "Hello from JS!" } ], temperature: 0.7 }) }; // Execution fetch(url, requestOptions) .then(res => res.json()) .then(data => { console.log(data.choices[0].message.content); }) .catch(error => console.error("Error:", error));

Common Errors

While we strive for 99.9% uptime, errors can occur due to network instability, invalid input formatting, or rate limiting. The Orgteh API utilizes standard HTTP status codes.

Below is a detailed breakdown of the most common errors you might encounter, along with actionable steps to resolve them.

401

Unauthorized

Authentication failure. Check your API key format and validity.

Solution: Verify your API key starts with "Orgteh_" and is active in your dashboard.
429

Too Many Requests

You have exceeded your daily limit. Wait for the reset or upgrade your plan.

Solution: Check your usage in the profile page or upgrade your plan.
400

Bad Request

Invalid request format or missing required parameters.

Solution: Verify your JSON structure and required fields (model, messages).
500 / 503

Server Errors

Transient server issue. Please retry with exponential backoff.

Solution: Wait a few seconds and retry. If persistent, contact support.

Need Help? For persistent issues, please share the x-request-id with our support team via the contact page.
